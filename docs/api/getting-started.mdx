# Connect API

The API allows us to interact with the blockchain and make queries via Javascript. The basic API is implemented on the Substrate layer and is the same for all substrate-based networks.

Complete API overview and cookbook is on the [Polkadot documentation portal](https://polkadot.js.org/docs/).

## Installation

```sh
npm install @gear-js/api
```

or

```sh
yarn add @gear-js/api
```

## Getting started

Start the API connection to the local running RPC node:

```javascript
import { GearApi } from '@gear-js/api';

const gearApi = await GearApi.create();
```

You can also connect to a different node:

```javascript
const gearApi = await GearApi.create({
  providerAddress: 'wss://someIP:somePort',
});
```

:::note

Below are a few entry points for interact with Gear RPC Node.

For connection to local node use:

```bash
ws://127.0.0.1:9944
```

For connection to Testnet use:

```bash
wss://rpc-node.gear-tech.io:443
```

:::


Getting node info

```javascript
const chain = await gearApi.chain();
const nodeName = await gearApi.nodeName();
const nodeVersion = await gearApi.nodeVersion();
const genesis = gearApi.genesisHash.toHex();
```

## Example

This simple example describes how to subscribe to a new blocks and get chain spec:

```js
async function connect() {
  const gearApi = await GearApi.create();

  const [chain, nodeName, nodeVersion] = await Promise.all([
    gearApi.chain(),
    gearApi.nodeName(),
    gearApi.nodeVersion(),
  ]);

  console.log(
    `You are connected to chain ${chain} using ${nodeName} v${nodeVersion}`,
  );

  const unsub = await gearApi.gearEvents.subscribeToNewBlocks((header) => {
    console.log(
      `New block with number: ${header.number.toNumber()} and hash: ${header.hash.toHex()}`,
    );
  });
}
connect().catch(console.error);
```
